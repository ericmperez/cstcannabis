## Compromiso con la accesibilidad

La Comisión para la Seguridad en el Tránsito de Puerto Rico (CST) se compromete a garantizar la accesibilidad digital para todas las personas, incluyendo personas con discapacidades. Nos esforzamos continuamente por mejorar la experiencia de usuario de este portal y aplicar los estándares de accesibilidad pertinentes.

## Estándar de conformidad

Este sitio web ha sido diseñado para cumplir con el nivel de conformidad **AA** de las **Pautas de Accesibilidad para el Contenido Web (WCAG) 2.1** del World Wide Web Consortium (W3C), de acuerdo con los requisitos de la **Ley 229-2003** del Estado Libre Asociado de Puerto Rico y la **Sección 508** de la Ley de Rehabilitación de los Estados Unidos.

## Medidas de accesibilidad implementadas

### Navegación
- Enlace de salto al contenido principal ("Ir al contenido principal")
- Estructura de encabezados jerárquica y semántica
- Migas de pan (breadcrumbs) en todas las páginas internas
- Menús de navegación identificados con roles ARIA

### Contenido
- Texto alternativo descriptivo en todas las imágenes informativas
- Relación de contraste mínima de 4.5:1 para texto normal
- Relación de contraste mínima de 3:1 para texto grande y componentes de interfaz
- Contenido disponible en español e inglés
- Atributo `lang="es"` en el documento

### Interactividad
- Todos los elementos interactivos son accesibles mediante teclado
- Indicadores de foco visibles (3px sólido) en todos los elementos enfocables
- Áreas de toque mínimas de 44x44 píxeles
- Mensajes de estado anunciados a lectores de pantalla (aria-live)
- Trampa de foco (focus trap) en ventanas de diálogo
- Cierre con tecla Escape en menús y diálogos

### Compatibilidad
- Modo de alto contraste soportado
- Colores forzados de Windows soportado
- Preferencia de movimiento reducido respetada
- Compatible con lectores de pantalla (NVDA, JAWS, VoiceOver)
- Compatible con navegación por teclado únicamente
- Diseño responsivo desde 320px de ancho

### Formularios
- Todos los campos tienen etiquetas asociadas
- Campos obligatorios marcados con `aria-required`
- Errores de validación anunciados automáticamente

## Tecnologías utilizadas

- HTML5 semántico
- CSS3 con propiedades personalizadas (custom properties)
- JavaScript (ES5+) con mejoras progresivas
- WAI-ARIA 1.2
- SVG para iconografía

## Evaluación

La accesibilidad de este sitio se evalúa mediante:
- Pruebas automáticas con axe DevTools y WAVE
- Navegación exclusiva por teclado
- Pruebas con lectores de pantalla (VoiceOver en macOS)
- Verificación manual de relaciones de contraste
- Pruebas en dispositivos móviles

## Limitaciones conocidas

A pesar de nuestros esfuerzos, algunas áreas pueden presentar limitaciones:

1. **Contenido de terceros**: Videos embebidos de YouTube pueden no tener subtítulos en español.
2. **Documentos PDF**: Algunos documentos descargables pueden no estar completamente etiquetados para lectores de pantalla.
3. **Chatbot**: Las respuestas del asistente virtual contienen HTML simple que puede no ser anunciado con formato completo por todos los lectores de pantalla.
